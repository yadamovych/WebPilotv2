# Extension Lint Fixer Agent

This agent automatically detects and fixes ESLint errors in the WebPilot extension, similar to the manual process performed for GitHub issue #3.

## Capabilities

- **Automatic Error Detection**: Scans extension files for ESLint violations
- **Intelligent Fixing**: Applies appropriate fixes:
  - Removes unused variables/functions
  - Adds eslint-disable comments for debug logging
  - Fixes regex escape sequences
  - Corrects loose equality operators
  - Proper module export checks
- **Verification**: Confirms all fixes pass linting
- **Issue Management**: Automatically closes resolved issues
- **GitHub Integration**: Works with GitHub Actions or manual invocation

## How It Works

### Trigger Points

1. **GitHub Issue**: When an issue is labeled with `extension-errors` and `automated`
2. **Manual Trigger**: Via `gh workflow run` or GitHub Actions UI
3. **Scheduled**: Can be set to run on a schedule (configure in workflow)

### Process Flow

```
1. Checkout code
   ↓
2. Install dependencies
   ↓
3. Run: npm run lint:fix
   ↓
4. Verify linting passes
   ↓
5. Commit changes (if any)
   ↓
6. Push to main
   ↓
7. Close issue / Add comment
```

## Usage

### GitHub Actions (Automatic)

The workflow triggers automatically when:
- A new issue is created with the `extension-errors` label
- An existing issue with `extension-errors` is edited

### Manual Trigger

```bash
gh workflow run auto-fix-extension-errors.yml
```

Or via GitHub Actions UI:
1. Go to Actions → "Auto-fix Extension Errors"
2. Click "Run workflow"

### File: `.github/workflows/auto-fix-extension-errors.yml`

Key configuration:
- **Triggers**: Issues with `extension-errors` label or manual dispatch
- **Node version**: 18.x
- **Auto-fix command**: `npm run lint:fix` in extension directory
- **Auto-commit**: Commits to main if changes are made
- **Issue Management**: Auto-closes if successful, adds comments for status

## Files Modified

The agent automatically fixes issues in:
- `extension/background.js`
- `extension/content.js`
- `extension/error-handler.js`
- `extension/popup.js`
- `extension/*.js` (any JS file with linting issues)

## Error Handling

- **If fixes pass**: Issue automatically closed with success comment
- **If fixes fail**: Issue gets a warning comment for manual review
- **If no errors**: Issue closed as not-needed
- **On error**: Workflow logs available for debugging

## Customization

Edit `.github/workflows/auto-fix-extension-errors.yml` to:
- Change target branch (currently `main`)
- Add additional linting steps
- Modify commit message
- Adjust permissions
- Add scheduled runs (cron)
